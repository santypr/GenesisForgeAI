# Global Constitution

## Mandatory Rules
1. Always document decisions in `/docs/architecture/decisions.md`.
2. Always ask for missing information before irreversible generation.
3. Always update `/docs` when code, architecture, or scope changes.
4. Always create task folders in `/docs/tasks/<task-id>/`.
5. Always maintain consistency with existing architecture and naming.

## Quality Rules
- Produce deterministic, composable outputs.
- Prefer explicit contracts over assumptions.
- Keep generated artifacts reproducible and idempotent.
